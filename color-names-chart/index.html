What Are HTML Color Names?

HTML color names are predefined keywords that represent specific colors in web design and development. Instead of using numeric values like HEX codes (#FF0000) or RGB colors (rgb(255, 0, 0)), developers can simply use the keyword Red. These CSS color names are part of the official specification and are parsed by every browser's rendering engine into their equivalent numeric values behind the scenes.

There are currently 148 standard color names supported by all modern browsers. The original set of 16 basic colors — including Red, Blue, Green, Black, White, and Yellow — was defined in the HTML 3.2 specification. The extended list of 148 names was later formalized in the CSS Color Module and includes evocative names like "Tomato", "DodgerBlue", "PeachPuff", and "RebeccaPurple" (added as a tribute to Rebecca Alison Meyer).

These named colors are not just convenient aliases — they are an integral part of the CSS specification documented by MDN Web Docs. Using them in your stylesheets ensures full cross-browser compatibility without any polyfills or fallbacks.

How Many Color Names Exist in CSS?

The CSS specification defines exactly 148 named colors. This number has remained stable since CSS Color Module Level 3. The original HTML standard included only 16 basic colors: Aqua, Black, Blue, Fuchsia, Gray, Green, Lime, Maroon, Navy, Olive, Purple, Red, Silver, Teal, White, and Yellow. The remaining 132 names were inherited from the X11 color system used in early Unix-based graphical interfaces.

While 148 may seem limited compared to the 16.7 million colors expressible through HEX or RGB values, these named colors cover a surprisingly wide and useful range. From warm tones like Sienna and Chocolate to cool tones like SteelBlue and CadetBlue, the named palette provides a solid foundation for web design. Difference Between HEX, RGB, and Named Colors

All three formats — HEX colors, RGB colors, and named colors — ultimately describe the same thing: a specific color displayed on screen. The difference lies in syntax, precision, and usability. Understanding when to use each format is essential for efficient web development.

HEX Colors

HEX (hexadecimal) codes are six-character strings prefixed with a hash symbol, like #FF6347. Each pair of characters represents the red, green, and blue channels in base-16 notation. HEX colors can express any of the 16,777,216 possible RGB colors, making them the most precise option for brand colors and production work. They are the most widely used format in design tools like Figma and Sketch.

RGB Colors

RGB colors use the rgb(r, g, b) function syntax, where each value ranges from 0 to 255. For example, rgb(255, 99, 71) is equivalent to #FF6347. The modern variant rgba() adds an alpha channel for transparency, which is useful for overlays and glassmorphism effects. Need to switch formats? Use our color converter for instant translation.

Named Colors

Named colors are the 148 human-readable keywords like "Tomato", "Coral", and "SkyBlue". They map one-to-one with specific HEX and RGB values. While they offer less variety than HEX or RGB, their strength lies in readability and developer experience. Writing color: DarkCyan; communicates intent far more clearly than color: #008B8B;.

In practice, many teams use named colors during prototyping and switch to HEX or RGB values in production for precision. Our contrast checker tool works with all three formats to help you ensure your text meets WCAG accessibility standards.

When to Use Named Colors in Design

Named colors shine in specific scenarios where speed, readability, and communication matter more than pixel-perfect precision. Here are the best use cases for HTML color names in your workflow:

  • Rapid Prototyping: When sketching out layouts and wireframes, typing background: Lavender; is faster than looking up a HEX code. You can always refine colors later using a color picker from image tool.
  • Team Communication: In code reviews and documentation, named colors are self-documenting. A teammate instantly understands border: 1px solid Gainsboro; without needing a visual reference.
  • Learning and Teaching: For beginners learning CSS, named colors provide an approachable entry point before diving into numeric color systems.
  • Theming and Dark Modes: Semantic color names in CSS variables like --surface: WhiteSmoke; make theme switching intuitive. For dark themes, explore our dark color palettes for inspiration.
  • Quick Fixes and Hotfixes: When deploying an urgent fix at 2 AM, color: Crimson; gets the job done without opening a design tool.

For production-grade work where exact color matching is required — such as brand guidelines or design system tokens — HEX and RGB values remain the recommended choice. Named colors are best used as a complementary tool in your design toolkit alongside gradient generators and palette builders.

Why Use Named Colors in CSS?

  • Readability: Code like color: Tomato; is easier to understand at a glance than color: #FF6347;, especially in large stylesheets with hundreds of declarations.
  • Memorability: It is easier to remember names like "SkyBlue", "ForestGreen", or "Chocolate" than their corresponding hex codes. This reduces context-switching between your editor and a color reference.
  • Speed: Typing a familiar color name is faster than looking up or recalling a six-digit hex value, which accelerates development during sprints and hackathons.
  • Transparency: You know exactly what color you are using just by reading the name. There is no ambiguity about whether #808000 is olive or lime — writing Olive makes it explicit.
  • Zero Dependencies: Named colors work everywhere — no preprocessors, no configuration files, no build steps required. They are native to CSS.

Frequently Asked Questions

How many HTML color names are there?

There are 148 standard HTML color names supported by all modern browsers. These were originally derived from the X11 color standard and later adopted into the CSS Color Module Level 3 specification by the W3C. The list includes 16 basic colors like Red, Blue, and Green, plus 132 extended names like CornflowerBlue, RebeccaPurple, and BurlyWood.

Can I use HTML color names in CSS?

Yes, all 148 HTML color names are valid CSS color values. You can use them directly in any CSS property that accepts a color, such as color: Tomato; or background-color: SlateBlue;. They work in all modern browsers and are part of the official CSS specification.

What is the difference between HEX and color names?

HEX codes like #FF0000 are six-digit hexadecimal numbers representing RGB channels, while color names like "Red" are human-readable keywords that map to those same HEX values. Both produce identical colors in the browser. Names are easier to remember and read in code, while HEX codes allow you to specify any of the 16.7 million possible RGB colors, not just the 148 named ones.

Are HTML color names case-sensitive?

No, HTML and CSS color names are not case-sensitive. You can write Tomato, tomato, or TOMATO and they will all render the same color. However, the convention is to use CamelCase for readability, which is also how they are documented on MDN and the W3C specification.

Why should I use named colors instead of HEX codes?

Named colors improve code readability. Writing color: Coral; is immediately understandable, while color: #FF7F50; requires a mental lookup. Named colors are ideal for prototyping, quick edits, and working in teams where clarity matters. For production work requiring precise brand colors, HEX or RGB values are still recommended.
