UI Design

Notification Colors

The visual language of alerts and system updates.

Notifications are the heartbeat of an application. They confirm actions, warn of issues, and provide updates. Because they appear contextually and briefly, their color coding must be instantly recognizable. A user shouldn't have to read the text to know if a notification is good news or bad news; the color should tell the story.

The Standard Color Codes

Over decades of computing, a standard has emerged. Green signifies success (e.g., "File saved"). Red signifies error or critical failure. Yellow/Orange signifies a warning (e.g., "Storage full"). Blue signifies neutral information (e.g., "New feature available"). Sticking to this code reduces cognitive load, as users instinctively understand the message's intent.

Toast Notifications vs. Badges

"Toasts" are temporary pop-ups. They should use soft backgrounds with the corresponding accent color for the icon/border. A bright solid red toast can be too distracting. "Badges" (like the red dot on an app icon) should use high-saturation colors to catch the peripheral vision.

Conclusion

Consistency is the golden rule of notifications. If you use green for "success" today, do not use it for "warning" tomorrow. Build a color system that users can trust.
